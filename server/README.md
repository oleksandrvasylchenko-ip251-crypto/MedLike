# MedLike Server (Node.js + Express + MySQL + Telegram Bot)

## Статус: заготовка (scaffold), потребує налаштування

Це повноцінна структура бекенду з п.7 та п.8 чек-листа аудиту — код
робочий і синтаксично перевірений (`node --check` пройдено для всіх
файлів), але **він не підключений до реальної бази даних чи
Telegram-бота**, тому що для цього потрібні ваші власні:

- MySQL-сервер (хост, логін, пароль, назва БД);
- Telegram Bot Token (створюється через **@BotFather**) та ID чату
  адміністратора, куди приходитимуть заявки.

Без цих даних коди/логіка написані правильно, але запустити й
"побачити в дії" я не можу — авторизуватись у вашій майбутній базі
даних чи Telegram-акаунті ззовні неможливо і не повинно бути можливим
з міркувань безпеки.

## Структура

```
server/
├── app.js                     # точка входу Express
├── package.json
├── .env.example                # приклад змінних середовища — скопіюйте в .env
├── config/
│   ├── db.js                   # пул з'єднань MySQL
│   ├── schema.sql               # SQL-схема таблиць (виконати один раз)
│   ├── telegram.js              # ініціалізація бота, надсилання сповіщень
│   └── telegramCallbacks.js     # обробка кнопок "Підтвердити/Скасувати"
├── controllers/
│   ├── bookingController.js
│   └── callRequestController.js
├── models/
│   ├── bookingModel.js
│   └── callRequestModel.js
├── middleware/
│   └── validate.js              # серверна валідація вхідних даних
├── routes/
│   ├── bookingRoutes.js         # POST /api/booking
│   └── callRequestRoutes.js     # POST /api/call-request
└── public/                      # (зарезервовано, наразі не використовується)
```

## Як запустити (коли будуть готові MySQL і Telegram Bot)

1. Встановити залежності:
   ```bash
   cd server
   npm install
   ```
2. Скопіювати `.env.example` у `.env` і заповнити реальними даними:
   ```bash
   cp .env.example .env
   ```
3. Створити базу даних і таблиці:
   ```bash
   mysql -u root -p < config/schema.sql
   ```
4. Запустити сервер:
   ```bash
   npm start
   ```
5. Перевірити: `GET http://localhost:3000/api/health` має повернути `{"ok":true, ...}`.

## Що вже "розмовляє" з фронтендом

У `script/api.js` на фронтенді вже підготовлені функції
`submitBooking()` і `submitCallRequest()`, які звертаються до
`/api/booking` та `/api/call-request`. Зараз вони закоментовані й
працюють у режимі "заглушки" (просто виводять дані в консоль і
показують той самий alert, що й раніше). Коли бекенд буде розгорнутий
і доступний за реальною адресою — розкоментуйте `fetch(...)` у
`script/api.js` (позначено `// TODO`) і вкажіть правильний
`API_BASE_URL`.

## Telegram-бот: що він уміє

- Надсилає адміністратору повідомлення про нову заявку на запис
  (ім'я, телефон, послуга, побажання) з кнопками "✅ Підтвердити" /
  "❌ Скасувати".
- Надсилає повідомлення про заявку на зворотній дзвінок.
- При натисканні кнопки — оновлює статус запису в MySQL
  (`confirmed` / `cancelled`) і відповідає в чат підтвердженням.

## Важливо: чому це не "живий" бекенд прямо зараз

Я навмисно не намагався підключити чи "вигадати" реальні креденшели
бази даних або токен бота — це були б нефункціональні або небезпечні
заглушки. Структура, SQL-схема та вся логіка готові й перевірені на
коректність синтаксису; лишається підключити ваші реальні MySQL і
Telegram Bot Token, після чого система запрацює "з коробки".
